# ແຜນຖ່າຍວິດີໂອ Demo — ລະບົບຈັດການໂຮງແຮມ (Hotel Management System)

> ຄວາມຍາວເປົ້າໝາຍ: **2–3 ນາທີ** (120–180 ວິນາທີ)
> ຈຸດປະສົງ: ສະແດງໃຫ້ເຫັນວ່າລະບົບເຮັດວຽກໄດ້ຄົບວົງຈອນ (end-to-end) ຕັ້ງແຕ່ຈອງຫ້ອງ → ເຊັກອິນ → ອອກບິນ → ຮັບເງິນ → ລາຍງານ

---

## 1. ກຽມຄວາມພ້ອມກ່ອນຖ່າຍ (Pre-production Checklist)

- [ ] ເປີດ dev server: `bun dev` (ເຂົ້າທີ່ `http://localhost:3000`)
- [ ] ໂຫຼດຂໍ້ມູນຕົວຢ່າງ: `bun run seed:hotel` ແລະ `bun run seed:billing`
- [ ] ກຽມ 3 ບັນຊີຜູ້ໃຊ້ໄວ້ສະແດງສິດ (RBAC):
  - **Admin** (ເຈົ້າຂອງ/ຜູ້ດູແລ) — ເຫັນທຸກຢ່າງ
  - **Receptionist** (ພະນັກງານຕ້ອນຮັບ) — ຈອງ/ເຊັກອິນ/ອອກບິນ
  - **Housekeeping** (ແມ່ບ້ານ) — ປ່ຽນສະຖານະຫ້ອງ
- [ ] ຄວາມລະອຽດໜ້າຈໍ **1920×1080**, ໂປຣແກຣມບັນທຶກໜ້າຈໍ (OBS / ScreenRec)
- [ ] ປິດການແຈ້ງເຕືອນ ແລະ tab ອື່ນໆ ທີ່ບໍ່ກ່ຽວຂ້ອງ
- [ ] ຊູມ browser ປະມານ 100–110% ໃຫ້ໂຕໜັງສືອ່ານງ່າຍ
- [ ] ກຽມ cursor ໃຫ້ເຫັນຊັດ (highlight click ຖ້າມີ)

## 3. ບົດ (Script) ລະອຽດ ແຕ່ລະສາກ

> ໝາຍເຫດ: ຂໍ້ຄວາມໃນ **"ບັນຍາຍ"** ແມ່ນສຳລັບ voice-over ຫຼື ຄຳບັນຍາຍໃຕ້ຈໍ.
> ບ່ອນ **"ການກະທຳໃນຈໍ"** ແມ່ນສິ່ງທີ່ຕ້ອງຄລິກ/ສະແດງໃຫ້ກົງກັບສຽງ.

### ສາກ 1 — ເປີດ (0:00 – 0:15)
- **ການກະທຳໃນຈໍ:** ສະແດງ logo / ຊື່ລະບົບ ແລະ ໜ້າ Dashboard ພາບລວມ
- **ບັນຍາຍ:**
  > "ນີ້ແມ່ນ **ລະບົບຈັດການໂຮງແຮມ** ທີ່ຊ່ວຍໃຫ້ໂຮງແຮມເຮັດວຽກໄດ້ຄົບວົງຈອນ — ຕັ້ງແຕ່ການຈອງຫ້ອງ, ເຊັກອິນ-ເຊັກເອົາ, ອອກບິນ-ຮັບເງິນ ໄປຈົນເຖິງລາຍງານສະຫຼຸບ. ມາເບິ່ງກັນວ່າມັນເຮັດວຽກແນວໃດ."

### ສາກ 2 — ເຂົ້າສູ່ລະບົບ + ສິດການໃຊ້ງານ (0:15 – 0:30)
- **ການກະທຳໃນຈໍ:** ສະແດງໜ້າ Login → ເຂົ້າດ້ວຍ Admin → ຊີ້ໃຫ້ເຫັນເມນູຂ້າງ
- **ບັນຍາຍ:**
  > "ລະບົບແບ່ງສິດເປັນ 3 ບົດບາດ: **ຜູ້ດູແລ, ພະນັກງານຕ້ອນຮັບ ແລະ ແມ່ບ້ານ**. ແຕ່ລະຄົນຈະເຫັນສະເພາະໜ້າທີ່ຕົນມີສິດ ດ້ວຍລະບົບ RBAC."

### ສາກ 3 — ຈັດການຫ້ອງພັກ (0:30 – 0:50)
- **ການກະທຳໃນຈໍ:**
  1. ເຂົ້າໜ້າ **ປະເພດຫ້ອງ (Room Types)** — ສະແດງລາຄາ/ຄວາມຈຸ
  2. ເຂົ້າໜ້າ **ຫ້ອງພັກ (Rooms)** — ສະແດງສະຖານະຫ້ອງ (ວ່າງ / ມີແຂກ / ກຳລັງອານາໄມຄວາມສະອາດ / ປິດສ້ອມແປງ)
- **ບັນຍາຍ:**
  > "ເຮົາສາມາດກຳນົດ **ປະເພດຫ້ອງ ແລະ ລາຄາ** ໄດ້, ແລະເບິ່ງ **ສະຖານະຫ້ອງແບບ real-time** ວ່າຫ້ອງໃດວ່າງ ຫ້ອງໃດມີແຂກພັກຢູ່."

### ສາກ 4 — ສ້າງລູກຄ້າ + ຈອງຫ້ອງ (0:50 – 1:20)
- **ການກະທຳໃນຈໍ:**
  1. ສ້າງ **ໂປຣໄຟລ໌ລູກຄ້າ (Guest)** ໃໝ່ — ໃສ່ຊື່, ເບີໂທ, ເລກເອກະສານ
  2. ສ້າງ **ການຈອງ (Reservation)** — ເລືອກລູກຄ້າ, ຫ້ອງ, ວັນເຂົ້າ-ອອກ
  3. ເປີດ **ປະຕິທິນຈອງ (Booking Calendar)** — ສະແດງການຈອງທີ່ຫາກໍສ້າງ
- **ບັນຍາຍ:**
  > "ມາລອງສ້າງລູກຄ້າໃໝ່ ແລ້ວເຮັດການຈອງ. ລະບົບຈະ **ກວດສອບຫ້ອງວ່າງ** ໂດຍອັດຕະໂນມັດ ປ້ອງກັນການຈອງຊ້ອນກັນ. ການຈອງຈະປະກົດຂຶ້ນເທິງ **ປະຕິທິນ** ໃຫ້ເຫັນພາບລວມໄດ້ທັນທີ."

### ສາກ 5 — ໜ້າຮັບແຂກ: ເຊັກອິນ / ເຊັກເອົາ (1:20 – 1:45)
- **ການກະທຳໃນຈໍ:**
  1. ເຂົ້າໜ້າ **Front Desk** → ຄົ້ນຫາການຈອງ → ກົດ **ເຊັກອິນ**
  2. ສະແດງໃຫ້ເຫັນວ່າສະຖານະຫ້ອງປ່ຽນເປັນ **"ມີແຂກພັກ"** ໂດຍອັດຕະໂນມັດ
  3. (ສືບຕໍ່) ກົດ **ເຊັກເອົາ** → ຫ້ອງປ່ຽນເປັນ **"ກຳລັງອານາໄມຄວາມສະອາດ"**
- **ບັນຍາຍ:**
  > "ເມື່ອແຂກມາຮອດ ກົດ **ເຊັກອິນ** — ສະຖານະຫ້ອງປ່ຽນເປັນມີແຂກພັກທັນທີ. ເມື່ອແຂກອອກ ກົດ **ເຊັກເອົາ** — ຫ້ອງຈະຖືກສົ່ງເຂົ້າຄິວອານາໄມໃຫ້ແມ່ບ້ານ ໂດຍບໍ່ຕ້ອງປ່ຽນເອງ."

### ສາກ 6 — ອອກບິນ + ຮັບຊຳລະ + ພາສີ (1:45 – 2:20)
- **ການກະທຳໃນຈໍ:**
  1. ສ້າງ **ໃບບິນ (Invoice)** ຈາກການເຂົ້າພັກ — ສະແດງ ຄ່າຫ້ອງ × ຈຳນວນຄືນ + ພາສີ + ຍອດລວມ
  2. ກົດ **ບັນທຶກການຊຳລະ (Payment)** — ເລືອກຊ່ອງທາງ (ເງິນສົດ / ໂອນ / ບັດ)
  3. ສະແດງສະຖານະບິນປ່ຽນເປັນ **"ຊຳລະແລ້ວ"** + (ຖ້າມີ) **QR ໃບບິນ**
- **ບັນຍາຍ:**
  > "ລະບົບອອກບິນໂດຍ **ຄິດໄລ່ຄ່າຫ້ອງ, ພາສີ ແລະ ຍອດລວມ** ໃຫ້ອັດຕະໂນມັດ. ຮັບເງິນໄດ້ຫຼາຍຊ່ອງທາງ ທັງເງິນສົດ, ໂອນ ແລະ ບັດ ພ້ອມຕິດຕາມຍອດຄ້າງຊຳລະ."

### ສາກ 7 — ຕັດກະເງິນສົດ (2:20 – 2:35)
- **ການກະທຳໃນຈໍ:** ເຂົ້າໜ້າ **Cash Shift** → ສະແດງ ເປີດກະ / ຍອດຮັບໃນກະ / ປິດກະ + ນັບເງິນຈິງ + ສ່ວນຕ່າງ (variance)
- **ບັນຍາຍ:**
  > "ໃນທ້າຍກະ ພະນັກງານສາມາດ **ປິດກະເງິນສົດ** — ລະບົບສະຫຼຸບຍອດຮັບແຍກຕາມຊ່ອງທາງ ແລະ ປຽບທຽບກັບເງິນຈິງທີ່ນັບໄດ້ ເພື່ອຄວາມໂປ່ງໃສ."

### ສາກ 8 — ລາຍງານ (2:35 – 2:50)
- **ການກະທຳໃນຈໍ:** ເຂົ້າໜ້າ **ລາຍງານ (Reports)** → ສະແດງ **ຍອດຂາຍລາຍວັນ** ແລະ **ອັດຕາການເຂົ້າພັກ (Occupancy)** ດ້ວຍກຣາຟ
- **ບັນຍາຍ:**
  > "ສຸດທ້າຍ ເຈົ້າຂອງສາມາດເບິ່ງ **ລາຍງານຍອດຂາຍລາຍວັນ ແລະ ອັດຕາການເຂົ້າພັກ** ເປັນກຣາຟ ເພື່ອປະເມີນຜົນການດຳເນີນງານ."

### ສາກ 9 — ສະຫຼຸບ + ປິດ (2:50 – 3:00)
- **ການກະທຳໃນຈໍ:** ກັບໄປໜ້າ Dashboard → ສະແດງ logo / ຂໍ້ຄວາມປິດ
- **ບັນຍາຍ:**
  > "ນີ້ຄືລະບົບຈັດການໂຮງແຮມທີ່ຄົບວົງຈອນ — ໃຊ້ງານງ່າຍ, ໄວ ແລະ ປອດໄພ. ຂອບໃຈທີ່ຮັບຊົມ."

## 4. ເຄັດລັບການຖ່າຍ (Production Tips)

- **ຄວນຮ່າງລຳດັບ click ກ່ອນ** — ບໍ່ໃຫ້ລັງເລ ຫຼື ຫາເມນູໃນຂະນະຖ່າຍ
- ຖ່າຍ **ແຕ່ລະສາກແຍກກັນ** ແລ້ວຄ່ອຍຕັດຕໍ່ ຈະງ່າຍກວ່າຖ່າຍຍາວເທື່ອດຽວ
- ໃຊ້ **ຂໍ້ມູນຕົວຢ່າງທີ່ສະອາດ** (ຊື່ລູກຄ້າ, ເລກຫ້ອງ ທີ່ອ່ານງ່າຍ) — ຫຼີກລ່ຽງຂໍ້ມູນຮົກ
- ເລັ່ງຄວາມໄວ (speed up) ໃນຊ່ວງທີ່ພິມຂໍ້ມູນຍາວ ເພື່ອປະຢັດເວລາ
- ໃສ່ **ຄຳບັນຍາຍໃຕ້ຈໍ (subtitle)** ເພື່ອໃຫ້ເຂົ້າໃຈໄດ້ແມ່ນບໍ່ມີສຽງ
- ໃສ່ **ດົນຕີປະກອບເບົາໆ** ໃນພື້ນຫຼັງ (ບໍ່ດັງເກີນສຽງບັນຍາຍ)
